## 一、2026年6月 国内可访问平台现状

### 海外平台在国内的访问情况（2026年6月确认）

| 平台 | 国内访问 | 是否需要备案 | 能否用于正式产品 |
|------|---------|-------------|----------------|
| **Vercel** (`*.vercel.app`) | ❌ DNS被污染，极不稳定 | 不需要 | ❌ 不推荐 |
| **Cloudflare Pages** | ❌ 基本无法访问 | 不需要 | ❌ 不推荐 |
| **GitHub Pages** | ⚠️ 时好时坏，经常超时 | 不需要 | ❌ 不推荐 |
| **Netlify** | ⚠️ 不稳定 | 不需要 | ❌ 不推荐 |

> **结论：所有主流海外托管平台（Vercel/Cloudflare/GitHub Pages/Netlify）在国内都不可靠，不能作为工业产品的正式部署方案。**
>
> Vercel 虽有 `cname-china.vercel-dns.com` 国内专属 CNAME 解析，但实际体验仍不稳定，不适合面向客户的正式产品。

## 四、方案A详解：香港服务器全栈部署（推荐）

### 架构

```
客户浏览器/微信
       ↓
  自有域名 (如 product.com)
       ↓
  DNS → 香港服务器 IP
       ↓
  ┌──────────────────────────┐
  │     香港 VPS (Docker)     │
  │  LightNode / 硅云         │
  ├──────────────────────────┤
  │  Nginx (反向代理+SSL)     │
  │  ├─ /          → 官网前端 │
  │  ├─ /demo/    → 演示前端  │
  │  ├─ /api/     → 后端 API  │
  │  └─ PostgreSQL (演示DB)   │
  └──────────────────────────┘
```

### 快速部署

```bash
# 1. 登录服务器
ssh root@你的香港IP

# 2. 一键安装 Docker
curl -fsSL https://get.docker.com | bash

# 3. 克隆你的项目
git clone <你的仓库> /opt/myproduct
cd /opt/myproduct

# 4. 一键启动
docker-compose up -d

# 5. 配置 Nginx + SSL（可选自动脚本）
# 完成！国内访问 30-80ms
```

### docker-compose.yml 示例

```yaml
version: '3.8'
services:
  web:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - api
    restart: always

  api:
    build: ./backend
    ports:
      - "3000:3000"
    environment:
      - DATABASE_URL=postgresql://user:pass@db:5432/demo
    depends_on:
      - db
    restart: always

  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: demo
      POSTGRES_USER: user
      POSTGRES_PASSWORD: pass
    volumes:
      - pgdata:/var/lib/postgresql/data
    restart: always

volumes:
  pgdata:
```

## 七、2026年 ICP 备案要点

### 2026年最新政策变化

- **审查更严**：强调备案内容与实际运营一致性
- **备案类型选错**将直接影响后续上线和业务拓展
- **备案成功后**：网站底部必须悬挂备案号并链接至工信部官网
- **个人主体**：不可用于商业产品宣传
- **经营性服务**（在线收费）：需额外申请 ICP 许可证（EDI）

### 企业备案流程

```
1. 购买境内服务器（阿里云/腾讯云）
2. 通过服务商提交备案
3. 准备材料：营业执照、法人身份证、域名证书
4. 审核：1-20个工作日
5. 通过后 → 境内CDN → 最佳体验
```

> **建议：无论选择哪个方案，同步启动备案。备案期间境外服务正常运营。**
