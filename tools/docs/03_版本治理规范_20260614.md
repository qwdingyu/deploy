# 版本治理规范

## 版本策略

所有 ZL 自有 NuGet 包采用**统一版本**策略。即：同一批次发布的所有 ZL 包（PlcBase 系列 + IoT SDK 系列）共享一个版本号。

当前版本：**2.2.1** (2026-06-14 升级)

## 单一版本来源（Single Source of Truth）

版本号的权威来源是：

```
ZL.PlcBase/Directory.Build.props
  → <Version>2.2.1</Version>
```

所有其他位置的版本号均由工具自动同步，**不允许手动编辑**。

### 版本传播链

```
PlcBase/Directory.Build.props  <Version>   ← 唯一入口
        │
        ▼
zl-pipeline publish --local <version>
        │
        ├─→ dotnet pack -p:PackageVersion=<version>     ← 生产者包版本
        ├─→ ~/.nuget/local-feed/                        ← 本地分发
        ├─→ sync-consumers <version>                    ← 消费者 CPM 同步
        └─→ version-check <version>                     ← 一致性门禁
```

## 版本升级流程

**唯一合法方式：**

```bash
./deploy-fast.sh <new-version>
```

该脚本会自动完成：
1. 构建 PlcBase + iot-sdk 所有包
2. 复制到 `~/.nuget/local-feed/`
3. 同步所有消费者 CPM
4. 执行版本一致性门禁

禁止单独修改某个消费者的 `Directory.Packages.props` 中的 ZL 包版本。

## 命令职责划分

| 命令 | 用途 | 何时使用 |
|------|------|---------|
| `publish --local <version>` | 仅构建+本地 feed | 开发调试 |
| `deploy-fast.sh [version]` | 完整本地发布（含消费者同步和门禁） | **标准日常操作** |
| `sync-consumers <version>` | 手动同步消费者 CPM | 修复已漂移的消费者 |
| `version-check <version>` | 检查所有消费者版本一致性 | 发布前门禁 / 日常巡检 |
| `align-versions --source auto` | 从包源查询每个包的最新版本 | **未来独立发版时使用** |
| `publish <version>` | 完整流水线（含混淆+NuGet.org推送） | 正式发布到 NuGet.org |

## 禁止行为（Prohibited Actions）

| 行为 | 后果 |
|------|------|
| 手动修改消费者 `Directory.Packages.props` 中 ZL 包的 Version | 版本漂移 |
| 运行 `align-versions`（无 `--source` 参数） | 会从 NuGet.org 获取旧版本（1.1.0），导致降级 |
| 部分构建（只用 PlcBase 不用 iot-sdk 或反之） | 包不完整，消费者 restore 失败 |

⚠️ **`align-versions` 命令设计用于独立版本模式。在统一版本模式下，应使用 `sync-consumers` 替代。**

## 门禁机制

`version-check` 命令作为门禁，在以下时机自动触发：
- `deploy-fast.sh` 执行完毕后
- 版本不一致时脚本会以非 0 退出码终止

门禁规则：
- 遍历所有消费者的 `Directory.Packages.props`
- 检查所有的 `ZL.*` 和 `ProtocolGateway.*` 包版本是否全部 == 当前版本
- 有一个不匹配即失败

## NuGet 源配置

```
~/.nuget/NuGet/NuGet.Config           ← 全局配置（继承给所有项目）
  ├─ local-feed   → ~/.nuget/local-feed/    ← 最高优先级
  ├─ nuget.org    → https://api.nuget.org/v3/index.json
  └─ IotSdkLocal  → iot-sdk/artifacts       ← 回退源
```

消费者项目（UseThink.Iot、tmom）的 `NuGet.config` 为空，继承全局配置。

## 故障排查

| 症状 | 原因 | 修复 |
|------|------|------|
| `dotnet restore` 报 NU1102 | 2.2.0 包不在 local-feed 中 | 运行 `deploy-fast.sh 2.2.0` |
| 消费者 restore 到混合版本 | CPM 中版本不一致 | 运行 `sync-consumers <version>` + `version-check <version>` |
| `version-check` 失败 | 有人手动修改了消费者 CPM | 运行 `sync-consumers <version>` 自动修复 |

## 场景与用法对照

### 场景 1：日常开发，修改了 iot-sdk 代码

```bash
# 1. 修改代码后，快速构建并推送到 local-feed
cd /Users/dingyuwang/0-X/deploy/tools
./deploy-fast.sh 2.2.0

# 2. 使用方（如 UseThink.Iot）直接 restore
cd /Users/dingyuwang/0-X/UseThink.Iot/api
dotnet restore UseThink.Iot.Web/UseThink.Iot.Web.csproj
# → 自动命中 local-feed 中的 2.2.0 版本
```

**适用产品**：iot-sdk（所有 24 个项目）、PlcBase（所有 4 个项目）

### 场景 2：修改了 PlcBase 代码

```bash
# 与场景 1 相同命令，deploy-fast.sh 自动处理两个仓库
cd /Users/dingyuwang/0-X/deploy/tools
./deploy-fast.sh 2.2.0
```

**说明**：`deploy-fast.sh` 总是同时构建 PlcBase 和 iot-sdk。这是强制性的——部分构建会破坏版本一致性。

### 场景 3：版本升级（2.2.0 → 2.3.0）

```bash
# 1. 更新 PlcBase 版本声
vi /Users/dingyuwang/0-X/ZL.PlcBase/Directory.Build.props
# 修改 <Version>2.2.0</Version> → <Version>2.3.0</Version>

# 2. 用新版本号运行 deploy-fast
cd /Users/dingyuwang/0-X/deploy/tools
./deploy-fast.sh 2.3.0

# 3. 此时所有消费者 CPM 自动更新为 2.3.0
```

**禁止**：手动修改 iot-sdk/Directory.Packages.props 或任何消费者 CPM 中的版本号。

### 场景 4：发现版本漂移，修复消费者

```bash
# 检测哪些消费者有问题
cd /Users/dingyuwang/0-X/iot-sdk
zl-pipeline version-check 2.2.0

# 自动修复
zl-pipeline sync-consumers 2.2.0

# 再次验证
zl-pipeline version-check 2.2.0
```

### 场景 5：未来推送到 NuGet.org

```bash
# 完整流水线（含混淆 + NuGet.org 推送）
export NUGET_API_KEY=your-key
cd /Users/dingyuwang/0-X/iot-sdk
zl-pipeline publish 2.2.0

# 同步消费者 CPM
zl-pipeline sync-consumers 2.2.0
zl-pipeline version-check 2.2.0
```

### 场景 6：未来独立版本（不同包不同版本号）

```bash
# 从 local-feed 查询各包最新版本，对齐消费者
zl-pipeline align-versions --source local

# 从 NuGet.org 查询（包已推送公网后使用）
zl-pipeline align-versions --source nuget

# 自动选择来源：local-feed 优先，没有则查 NuGet.org
zl-pipeline align-versions --source auto
```

## 产品与命令对照表

| 产品/项目 | 工具命令 | 说明 |
|----------|---------|------|
| **ZL.PlcBase**（4 个包） | `deploy-fast.sh <version>` | 自动包含在 deploy-fast 中 |
| **iot-sdk**（24 个包） | `deploy-fast.sh <version>` | 自动包含在 deploy-fast 中 |
| **UseThink.Iot**（消费者） | `sync-consumers <version>` | 构建后自动同步 |
| **tmom**（消费者） | `sync-consumers <version>` | 构建后自动同步 |
| **所有消费者巡检** | `version-check <version>` | 构建后自动执行 |
| **本地开发调试** | `deploy-fast.sh`（默认 2.2.0） | 最常用命令 |
| **正式发布含混淆** | `zl-pipeline publish <version>` | 需要 NUGET_API_KEY |
